309: Step tracking game

In this project I and three other developers had the goal to create a step tracking game for Android. The idea was not only to encourage walking by making a game of it, but also learning how to develop for Android platforms. The game theme was that the user was pushing a lawnmower and each step went towards progress on mowing a lawn. The lawns would get progressively bigger but the user could purchase upgrades with money earned for completing lawns.


Team size:
3 (the fourth guy disappeared after the first month and as not been heard from since)


My role:
I was largely in charge of server communication. I used Volley to transfer user info to our database along with communicating friend requests to and from the user. I also had a share in creating the screens and simple navigation. I created a screen that took the history of the user's walking and displayed it in a bar graph. Lastly I worked with our main dev to create the architecture and game logic.


Technologies used:
Android, SQL, PHP, Volley, Android Studios, Genymotion Android Emulator, Trello, and Moqup.


What I learned:
I had previously only had experience in Java. I had never been on a development team or had any experience in architecture design. In this project I learned a lot about all of that along with all of the technologies listed. I learned about project flow, Agile development and what it takes to create a functional application.



329: Heathio

This project was an extension off of a Hack ISU project that we wanted to further develop. We got that chance in SE 329. The idea behind this one creating a system for a nurse or doctor to know the status of a patient in their hospital. The way we had it set up was there was a main server. A producer (A patient outputing heart rate, tempurture etc), and a consumer (the nurse or doctor). In the set up we would start the server, start the patient producing random values to simulate a patient, and then the consumer would connect and enter in a specific patient ID and the monitor that patients status along with viewing the records of said patient.


Team size:
4


My role:
I created the consumer page and did some work on the server communication.


Technologies used:
C9, Javascript, JQuery, HTML, CSS, SocketIO.


What I learned:
At this point I had some JS/html experience but in this I learned about server communication through web pages and dynamically injecting HTML with Javascript. Also updating HTML elements with JS was a really cool thing to learn.
